What is a Recurrence Relation?

A recurrence relation is an equation that describes the running time of a recursive algorithm in terms of the time it takes on smaller inputs. For example, imagine searching for a name in a sorted list of 1,000 entries. Instead of checking all 1,000, you look at the middle entry and throw away the half that can't contain the answer. How many steps does that take? The answer is captured by T(n) = T(n/2) + 1 — one comparison plus a recursive search on half the list.

Why Do We Care?

Recurrence relations let us figure out how fast (or slow) a recursive algorithm runs without actually running it. By solving the recurrence, we get the Big-O complexity — whether the algorithm scales gracefully like O(log n), reasonably like O(n log n), or catastrophically like O(2n).

How to Use This Tool

This tool has three modules, each showing a different way to analyze recurrences:

1. Recursion Tree — See how the algorithm breaks down. Each node is a recursive call. Watch the tree grow level by level and observe how work accumulates.
2. Master Theorem — A formula-based shortcut for divide-and-conquer recurrences. Compare two numbers and get the Big-O instantly.
3. Substitution Method — A formal proof technique. Guess the answer, then prove it correct step by step using mathematical induction.

The Big Picture: How Complexity Classes Compare

Same input size, vastly different running times:

n O(log n) O(n) O(n log n) O(2n)
10 3 10 33 1,024
20 4 20 86 1,048,576
30 5 30 147 1,073,741,824
50 6 50 282 1.13 × 1015
100 7 100 664 1.27 × 1030

At n=100, O(2n) is a number with 30 digits — more than the atoms in a human body.
